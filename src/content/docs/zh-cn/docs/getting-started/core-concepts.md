## 存储相关名词

### 对象存储

对象存储是一种通过网络保存文件的服务。图片、视频、压缩包等内容会以“对象”的形式保存，并通过 HTTP API 上传和下载。

它与电脑磁盘上的文件系统不同。对象存储通常没有真正的文件夹，界面中的目录是根据对象名称模拟出来的。它适合保存大量文件，也容易按使用量扩展容量。

### AWS S3

AWS S3 的全称是 Amazon Simple Storage Service，是亚马逊云提供的对象存储产品。S3 定义了 Bucket、Object、Key、Region 等概念，以及一套操作对象的 API。

只有明确写成 **AWS S3** 时，才特指亚马逊提供的这项服务。

### S3 API

S3 API 是 AWS S3 使用的接口约定，包括怎样上传、下载、列举和删除对象，怎样对请求签名，以及怎样生成预签名 URL。

大家常说的“S3 协议”，通常指的就是这套 API 约定。它不是像 HTTP 那样由独立标准组织定义的通用协议。

### S3 兼容对象存储

如果其他对象存储实现了常用的 S3 API，原本连接 AWS S3 的软件也可以用类似方式连接它，这类服务就叫“S3 兼容对象存储”。

Cloudflare R2、Backblaze B2、Tigris、MinIO 和 RustFS 都属于这一类。它们并不是 AWS S3，只是提供了兼容接口。

“S3 兼容”也不代表所有服务的每个细节都完全相同。ZPan 需要服务支持对象上传、下载、删除、列举、分片上传、预签名 URL 和浏览器 CORS。

### 泛指的 S3

在日常交流中，“使用 S3”经常泛指使用任意一种 S3 兼容对象存储，并不一定表示使用 AWS。

因此，ZPan 文档中的“S3”如果没有特别注明 AWS，通常都指整个 S3 兼容存储生态。把 ZPan 连接到 R2 或 MinIO，不会让文件经过或保存到亚马逊。

### Bucket

Bucket 是对象存储中保存对象的容器，可以简单理解为一个存储空间。例如，可以创建一个名为 `zpan-files` 的 Bucket，专门保存 ZPan 文件。

Bucket 名称、可用区域和命名规则由对象存储服务商决定。

### Object

Object 是对象存储中真正保存的一份内容。一个 Object 通常包含文件数据、Object Key 和少量元数据。

### Object Key

Object Key 是对象在 Bucket 中的唯一名称，例如：

```text
users/123/images/cover.png
```

这里的 `/` 只是名称的一部分。对象存储可以根据这些前缀展示出类似文件夹的效果，但它不一定对应真实磁盘目录。

### Endpoint

Endpoint 是对象存储 API 的访问地址。ZPan 使用它向存储服务发送请求，浏览器也可能通过它打开预签名上传或下载地址。

例如，AWS S3、Cloudflare R2 和自建 MinIO 都有不同格式的 Endpoint。使用浏览器直传时，这个地址必须能从用户的浏览器访问。

### Region

Region 表示 Bucket 所在的区域，例如 `us-east-1`。AWS S3 等云服务会使用 Region 选择数据中心并参与请求签名。

部分 S3 兼容服务并没有真正的多区域概念，但为了兼容 S3 API，仍然会要求填写一个固定值。

### Access Key 和 Secret Key

它们是一组访问对象存储的凭证：

- **Access Key** 类似用户名，用来标识调用方。
- **Secret Key** 类似密码，用来证明调用方拥有权限。

Secret Key 必须保密。它不是提供给普通用户的登录密码，也不应该写进公开文档、截图或前端代码。

### 预签名 URL

预签名 URL 是一个有有效期、带有临时授权的对象存储地址。拿到这个地址的浏览器可以在限定时间内上传或下载指定对象，不需要知道 Secret Key。

ZPan 使用预签名 URL 让文件尽可能直接在浏览器与对象存储之间传输。

### CORS

CORS 是浏览器的跨域访问规则。ZPan 页面和对象存储通常使用不同域名，因此 Bucket 必须允许 ZPan 的站点域名发起上传、下载或读取响应头。

CORS 配置错误时，ZPan 服务端可能可以连接 Bucket，但用户仍然会在浏览器中上传失败。

### Path-style

Path-style 是一种 S3 请求地址格式，会把 Bucket 名放在 URL 路径中。部分 MinIO、RustFS 或其他自建服务需要开启它；多数公有云服务使用默认格式即可。

## ZPan 相关名词

### 实例

一套已经部署并使用独立数据库的 ZPan 站点称为一个实例。它有自己的域名、用户、团队、设置和存储配置。

### 数据库

数据库保存账号、会话、文件名称、目录关系、Object Key、分享、配额和站点设置等信息。文件本身的内容保存在对象存储中，不保存在 ZPan 数据库里。

### 存储后端

站长在 ZPan 中添加的一套 Bucket、Endpoint、Region 和访问凭证称为一个存储后端。一个实例可以配置多个存储后端。

### Public URL

Public URL 是用户实际访问 ZPan 的完整公开地址，例如 `https://drive.example.com`。分享链接、登录回调、WebDAV 地址和部分公开资源都会以它为基础生成。

### 站长

站长是整个 ZPan 实例的管理员，可以配置对象存储、登录方式、站点信息和下载器，并管理用户、团队、公告与授权。

全新数据库中注册的第一个账号会成为站长。

### 个人空间

个人空间是每个用户自己的文件区域。这里的文件、目录和配额归该用户个人使用。

### 团队空间

团队空间是属于一个团队的独立文件区域。它有自己的成员、角色、文件和配额，不是某个用户个人空间中的普通文件夹。

### 团队角色

团队角色决定成员能否查看、修改或管理团队空间。团队所有者管理自己的团队，但不等于整个站点的站长。

## 功能相关名词

### 文件分享

文件分享是把文件或目录提供给空间之外的人访问。分享可以设置密码、有效期和下载次数限制，不需要把访问者加入团队。

### 图床

图床是一种专门用于上传和公开展示图片的服务。ZPan 图床使用已经配置的对象存储，并提供公开图片 URL、路径规则、图库管理和 API Key。

PicGo、PicList、uPic、ShareX、Flameshot 等工具可以把图片上传到 ZPan，并把返回的 URL 用于 Markdown、博客或论坛。

### WebDAV

WebDAV 是一套基于 HTTP 的远程文件访问协议。Finder、Windows 文件资源管理器、Cyberduck、RaiDrive 等客户端可以通过它访问 ZPan 文件。

WebDAV 客户端看到的是 ZPan 中的个人空间和团队空间，不是 Bucket 中的原始 Object Key，也不需要对象存储凭证。

### 远程下载

远程下载是让 ZPan 根据 HTTP 地址、磁力链接或种子创建下载任务。用户指定保存位置后，任务由可用下载器执行，完成的文件会被导入 ZPan。

### 下载器

下载器是实际执行远程下载任务的独立程序。它负责连接下载来源、保存临时文件、上报进度，并把结果上传到对象存储。

下载器需要持续运行，并使用自己的网络和临时磁盘。站长负责部署和授权下载器，普通用户只负责创建和管理自己的远程下载任务。

### API Key

API Key 是提供给程序和外部工具使用的访问凭证。它可以让脚本、图床工具或其他客户端调用 ZPan，而不需要保存用户的网页登录密码。

API Key 通常只在创建时显示一次。不同工具应使用不同的 Key，以便单独撤销。
