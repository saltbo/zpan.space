部署完成只代表程序已经运行。要让登录、分享、图床和 WebDAV 在正式域名下稳定工作，还需要完成一次站点级配置。

![ZPan 管理后台的站点设置](/images/docs/admin-settings.png)

## 先设置 Public URL

进入 **管理控制台 → 设置 → 站点标识**，填写最终 HTTPS 地址，例如 `https://files.example.com`。它必须是一个不带路径、查询参数和 Fragment 的 Origin。

Public URL 会参与生成分享链接、图床地址和 WebDAV 地址。OAuth Callback 则优先使用部署环境中的 `BETTER_AUTH_URL`；没有配置时才使用当前请求域名。生产环境应让这两个地址与用户访问域名一致。如果先用临时域名配置 OAuth，后来再换正式域名，第三方登录通常会因为 Callback 不一致而失败。

## 决定用户如何加入

新数据库中的首个注册用户会成为管理员。完成站点初始化后，应立即决定注册策略：

- 私人站点建议关闭开放注册，只通过管理员邀请添加用户。
- 对外服务需要 Pro 的开放注册能力，并应同时启用验证码、邮件验证和容量限制。
- 不要在没有反滥用措施时直接开放注册，否则匿名账号可能消耗存储和出口流量。

## 配置默认配额

默认个人配额和默认团队配额只影响新建空间。修改默认值不会自动重写现有用户已经获得的权益。需要调整单个用户或团队时，在对应详情页单独授权。

## 配置验证码

ZPan 支持 Cloudflare Turnstile、Google reCAPTCHA、hCaptcha 与 CaptchaFox。启用前先在服务商控制台添加最终站点域名，再填写 Site Key 与 Secret Key。

建议先保存配置并在无痕窗口验证注册、登录和找回密码，再强制开启。错误的域名或 Secret 会把正常用户挡在登录页之外。

## 配置邮件

邮件用于验证地址、邀请成员和找回密码。可选择 Cloudflare Email、SMTP 或兼容的 HTTP 邮件接口。填写发件人后先使用“发送测试邮件”，确认 SPF、DKIM 与垃圾邮件策略正常，再开启“要求邮箱验证”。

## 最后的站长检查

1. 用无痕窗口打开正式域名。
2. 验证注册入口符合预期。
3. 完成一次登录和退出。
4. 创建分享链接，确认域名正确。
5. 运行对象存储连接测试。
6. 如果启用邮件，完成一次真实收信。
