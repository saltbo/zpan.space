“文件”页面是浏览和整理当前个人或团队空间内容的主要工作区。

![ZPan 英文版文件管理界面](/images/docs/files.png)

大批量导入前先用 **新建文件夹** 建立稳定结构。ZPan 展示文件夹，底层对象存储可能使用对象键 Prefix 表示。用户不需要 Bucket 凭据，也不应绕过 ZPan 修改原始对象名。

选中文件后，可以根据权限执行下载、分享、移动、重命名、复制或移入回收站。文件夹操作可能涉及大量对象，完成前保持页面打开，并在重复操作前检查目标位置。

团队内容应通过空间选择器切换，不要用重复的顶层文件夹模拟团队。Breadcrumb 始终显示当前位置。找不到文件时，先确认当前空间、搜索条件和回收站，再请管理员检查存储。

避免多个浏览器 Tab 或 WebDAV 客户端同时执行冲突修改。重要批量操作应分批进行，并在结束后抽样下载验证。

## 上传与下载

传输通常通过短期签名 URL 在浏览器与对象存储之间直连。上传确认完成前保持 ZPan 页面打开，分片文件尤其如此。

上传失败后先查看第一个错误再重试。网络中断可以重试；配额、权限、CORS 或存储故障需要先修复。进度到 100% 但确认失败时，先刷新目录，不要立即重复上传副本。

下载是打开还是保存取决于文件类型和浏览器设置。对象存储签名 URL 会过期，不要收藏它，应重新从 ZPan 发起下载。

## 回收站与永久删除

删除会把支持的内容移入回收站。回收站只是恢复窗口，不是备份。恢复时目标位置必须可用，名称冲突可能要求更换目录或文件名。

永久删除会移除 ZPan 记录并请求对象存储删除，无法通过界面撤销。清空回收站前核对当前空间和选中项目。管理员可能执行自动清理，重要内容应尽快恢复。
