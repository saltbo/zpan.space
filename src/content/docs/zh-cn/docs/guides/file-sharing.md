---
title: 文件分享
description: 使用访问页、直链、密码、有效期和下载次数安全交付文件与文件夹。
---

分享是在现有文件上授予受控访问，不会复制一份对象，也不会暴露 Bucket 凭证。

给人发送文件时优先使用**访问页分享**。它可以展示文件信息，并支持密码、有效期、下载次数、文件夹浏览和保存到云盘。脚本、Release 或嵌入场景确实需要立即返回文件时，再使用**直链分享**。

## 创建并验证分享

1. 选中文件或文件夹并点击“分享”。
2. 除非接收方明确需要直链，否则选择访问页。
3. 非公开内容设置密码。
4. 设置满足交付需要的最短有效期；接收人数明确时增加下载次数限制。
5. 在没有登录 ZPan 的无痕窗口中测试链接。

文件夹分享保留 ZPan 的目录浏览，而不是直接暴露对象 Key。测试时应进入子目录并实际下载一个文件。

## 理解撤销边界

撤销会阻止后续授权，但无法收回已经下载的文件，也无法保证清除第三方缓存中的直链响应。直链更容易自动化，也更难控制传播范围。

定期在“我的分享”中清理过期交付链接并检查异常访问。撤销分享不会删除源文件。
