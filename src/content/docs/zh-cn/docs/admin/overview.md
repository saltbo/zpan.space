只有 `admin` 角色可以进入管理控制台。新数据库中的第一个注册账号会自动成为管理员；完成初始化后，不要再依赖“第一个用户”机制，而应通过后台管理后续账号。

![ZPan 管理后台概览](/images/docs/admin-overview.png)

## 推荐的首次配置顺序

1. 在“设置”中确认 Public URL 与注册策略。
2. 添加对象存储并完成浏览器直传测试。
3. 配置邮件，再决定是否要求邮箱验证。
4. 按需添加 OAuth 或 OIDC 登录。
5. 创建普通测试用户，验证配额、上传、分享和回收站。
6. 最后再启用公告、WebDAV、下载器或付费版能力。

## 左侧菜单分别负责什么

- **概览 / 数据分析**：查看用户增长、容量、流量、分享和后台操作趋势。
- **用户 / 团队**：管理账号状态、成员关系与存储权益。
- **存储**：配置 S3 后端、容量与连接测试。
- **下载器**：查看远程下载节点、心跳和任务能力。
- **OAuth / 设置**：管理登录提供商与全站策略。
- **站点公告**：向用户发布维护与产品消息。
- **审计日志**：追踪管理员和关键资源操作。
- **授权 / 关于**：绑定 Pro 或 Business，并检查版本和运行环境。

概览页的统计不是配置入口。发现容量或活跃度异常后，应回到对应的用户、团队、存储或审计页面定位原因。
