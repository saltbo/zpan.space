# ZPan documentation

ZPan is an open-source, S3-native file platform. It gives your object storage a modern file manager, image hosting, secure sharing, WebDAV, remote downloads, teams, and background tasks.

Files upload directly from clients to S3-compatible storage through presigned URLs. ZPan handles identity, metadata, permissions, and signing without relaying file traffic through the application server.

## Start here

1. [Choose a deployment](/docs/getting-started/choose-deployment/).
2. Complete the [Cloudflare Workers](/docs/deployment/cloudflare/) or [Docker](/docs/deployment/docker/) deployment guide.
3. Follow the [first-run setup](/docs/getting-started/first-run/) to connect S3-compatible storage and verify uploads.
4. Configure [file sharing](/docs/user-guide/file-sharing/) or [image hosting](/docs/user-guide/image-hosting/) for your workflow.

## Choose a deployment

Cloudflare Workers with D1 is the primary deployment target and requires no server maintenance. Docker uses the Node.js runtime and SQLite when you want to run ZPan on your own host or network.

ZPan also supports AWS, Vercel, Netlify, Azure, and Google Cloud. The application remains the same; only the platform adapter and database configuration change.

## Need a hosted version?

Open the [official ZPan Drive](https://drive.zpan.space) to use ZPan without deploying infrastructure. Pro and Business licenses are available through [ZPan Cloud](https://cloud.zpan.space).
