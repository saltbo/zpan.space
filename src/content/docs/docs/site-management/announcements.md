---
title: Site announcements
description: Publish clear, time-bounded notices for ZPan users.
sidebar:
  order: 3
---

Use announcements for maintenance windows, service incidents, policy changes, migrations, and important releases. They are a visible notice inside ZPan, not a replacement for durable documentation or an external status page.

![Site announcements in the English admin console](/images/docs/admin-announcement.png)

Write the outcome first: what is changing, who is affected, when it starts, and what users need to do. Use an absolute date, time, and timezone. Link to a longer document when the change needs procedures, screenshots, or a changelog.

Before publishing, verify links and preview the message on desktop and mobile. Avoid credentials, private incident data, internal hostnames, or details that could expose user information.

Update an incident announcement when the state changes instead of leaving contradictory notices active. Remove or archive expired announcements after the event so the interface retains urgency. For a planned maintenance window, publish before the event, update when work begins, and close it with the result and any remaining limitation.

Announcements are license-dependent. Administrative changes can be reviewed through [Audit logs](/docs/site-management/audit-logs/); delivery and readership should not be treated as guaranteed acknowledgment.
