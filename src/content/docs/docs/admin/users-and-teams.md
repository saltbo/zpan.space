Files belong to spaces rather than directly to user records. Every user has a personal space and may join team spaces, so administrators should confirm the target scope before granting capacity.

![The user list in the admin console](/images/docs/admin-users.png)

Disabling a user blocks sign-in without immediately deleting files. User deletion is irreversible; resolve team ownership and retained data first. Invite private users instead of sharing credentials.

Teams have independent members, roles, files, and quotas. Keep at least one active Owner. Viewer is appropriate for read-only access, while Editor manages files.

Defaults apply when a space is created. Existing spaces use explicit entitlements with capacity, source, note, and optional expiration. Record a reason for manual grants and review them through Audit Logs.
