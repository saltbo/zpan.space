对象存储便宜、可靠、容量几乎可以无限扩展，却很少有人愿意每天打开云厂商控制台管理文件。Bucket、Object Key 和 Access Key 是基础设施概念，不是普通用户的文件工作方式。

ZPan 做的事情很直接：在任意 S3 兼容存储之上提供文件目录、用户与团队、图床、安全分享、WebDAV 和远程下载，让对象存储拥有一个适合人使用、也适合站长运营的控制平面。

## 它不是把所有网盘挂在一起

ZPan 只支持 S3 兼容对象存储，不聚合百度网盘、Google Drive 或其他消费云盘账号，也不把服务器本地目录当作主要文件根。

这个边界减少了驱动差异。R2、S3、B2、Tigris、MinIO 与 RustFS 共享稳定的对象接口，ZPan 可以把精力放在文件工作流，而不是不断追逐第三方网盘的登录和接口变化。

如果你需要多人在线编辑文档、邮件、日历与企业协作套件，Nextcloud 一类产品更合适；如果你只想浏览服务器目录，File Browser 更直接；如果核心需求是聚合大量网盘提供商，AList 的方向更接近。

## 文件不经过应用服务器中转

ZPan 创建上传会话并签发预签名 URL，浏览器随后直接把文件传到 S3。下载也尽量交给对象存储。应用负责身份、权限、目录、配额和分享，不负责用自己的带宽搬运每一个字节。

这使 Cloudflare Workers 成为可行的主运行时：控制平面不需要一台高带宽 VPS，也不用让 NAS 永久在线。需要内网或本地主机时，Docker 仍然使用同一套产品能力。

## 四个核心使用场景

**S3 Web Drive** 为图片、视频、音乐和文档提供目录、搜索、预览、回收站与团队空间。

**图床**通过独立 API Key 接入 PicGo、PicList、uPic、ShareX 与 Flameshot，返回自己域名下的稳定链接。

**安全分享**提供访问页、直链、密码、有效期、下载次数、文件夹浏览和保存到云盘。

**外部文件工作流**通过 WebDAV 连接文件管理器和播放器，通过独立下载节点把远程文件直接导入对象存储。

## 自托管与官方托管

愿意管理域名、Bucket 和账号的用户可以部署 Community。核心文件能力保持 AGPL-3.0 开源。需要白标、开放注册、审计和更高运营能力时，可以购买 Pro；需要配额商品、Credits 和商业闭环时，则使用 Business。

不想维护基础设施的用户可以直接使用官方托管的 [ZPan Drive](https://drive.zpan.space)。托管是一种交付方式，不是第四个功能版本。

## 从一条完整路径开始

第一次体验不要急着开启所有功能。选择 Workers 或 Docker，注册管理员，设置 Public URL，添加 Bucket 并完成浏览器直传测试，再创建一个分享链接。基础链路跑通之后，再接入邮件、OAuth、WebDAV、图床或下载器。

先完成[部署前准备](/zh-cn/docs/getting-started/prerequisites/)，或者前往 [GitHub](https://github.com/saltbo/zpan) 查看代码与 Release。
