ZPan 2.0 是一次完整重写，也是当前 V2 产品线的起点。项目从 Go 迁移到 TypeScript，后端改用 Hono，前端使用 React；同一套应用可以运行在 Cloudflare Workers，也可以通过 Node.js 与 Docker 部署。

这次变化解决的并非单一功能，而是 ZPan 以后如何运行、如何扩展，以及文件管理能力如何继续演进。

## 一个应用，两种运行环境

V2 把业务逻辑建立在 Web 标准接口上。Cloudflare Workers 可以承担 Serverless 部署，Docker 则保留传统自托管方式，适合已有服务器、NAS 或本地网络环境的用户。

v2.0.1 随后补齐 Cloudflare Workers 的一键部署入口，并修复 `baseURL`、可信来源推断与 `nodejs_compat_v2` 配置。它让 v2.0.0 中已经具备的 Workers 架构真正变成可重复部署的方案。

部署位置改变后，用户文件仍不由应用服务器中转。ZPan 负责认证、权限、目录和上传会话，浏览器拿到预签名 URL 后直接连接 S3 兼容对象存储。应用承担控制逻辑，对象存储承担文件内容。

## 文件管理器重新实现

2.0 重做了日常文件操作，包括文件夹树、搜索和回收站。用户可以在统一界面中管理目录、查找文件，并在误删后从回收站恢复。

预览能力覆盖图片、PDF、代码、音频和视频。管理后台提供用户、存储与配额管理，组织可以拥有独立存储配额；界面同时支持英文和中文。

首个版本还修正了服务端全局搜索与媒体预览。全局搜索会在按下 Enter 后交给服务端处理，部分媒体文件无法正确渲染的问题也得到修复。

## 桌面之外的使用体验

同日发布的 v2.0.2 继续完善响应式布局。文件界面会适配桌面、平板和手机；移动端预览根据文件类型使用自适应底部抽屉，避免把桌面界面简单压缩到窄屏中。

因此，ZPan 2.0 主版本最终包含三个连续发布：

- **v2.0.0**：TypeScript 重写、双运行环境、S3 直传、文件管理与预览。
- **v2.0.1**：Cloudflare Workers 一键部署与认证配置修复。
- **v2.0.2**：桌面、平板和手机的响应式界面与移动预览。

## 升级说明

2.0 不是可以直接覆盖 V1 的普通补丁。迁移前应保留原数据库与对象存储备份，再按照 V2 的数据模型和部署文档重新配置实例。部署完成后，还需要在正式域名下实际验证登录、上传、下载和预览。
