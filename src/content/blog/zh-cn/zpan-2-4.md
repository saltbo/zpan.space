ZPan 2.4 在网盘之外加入独立的图床使用方式。图片可以在专门的图库中管理，通过稳定链接访问，也可以从常用截图和图片上传工具直接发送到 ZPan。

## 独立图库与上传路径

图床拥有自己的图库和设置页面，不需要把图片混在普通文件目录中管理。

上传支持两阶段模式与流式代理模式。不同存储和访问环境可以采用适合自己的路径，相关草稿状态也会在上传完成后正确处理。

## 自定义图片域名

管理员可以为图床配置自定义域名，并通过 Cloudflare for SaaS 管理域名接入。图片公开地址与 ZPan 主站地址可以分开配置。

这项更新同时统一公开文件链接到 `/r/:token`。依赖旧公开地址的页面或外部工具需要更新链接规则。

## PicGo、uPic 与 ShareX

2.4 提供 PicGo、uPic 和 ShareX 的配置，让用户可以在截图、编辑或整理图片后直接上传。程序化请求通过 API Key 认证，不需要把网页登录会话交给外部工具。

本次主版本修正了三款工具的配置生成、草稿图片过滤，以及大文件和分片上传问题。

## v2.4.1：Docker 修复

紧随主版本发布的 v2.4.1 修复 Docker 镜像在 `8222` 端口访问时返回 404 的问题，并简化镜像构建。使用 v2.4.0 Docker 镜像的实例应升级到该补丁。
